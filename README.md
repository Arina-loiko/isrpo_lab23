# Лабораторная работа 23 - Интерактивные веб-приложения

## Крестики-нолики

### Описание игры

Два игрока по очереди ставят X и O на поле 3x3. Побеждает тот, кто первым выстроит три своих символа в ряд - по горизонтали, вертикали или диагонали. Если все клетки заняты и никто не победил - ничья

### Правила игры

- Первым всегда ходит игрок X
- Игроки ходят по очереди
- Нельзя ставить символ в уже занятую клетку
- Игра заканчивается когда кто-то победил или наступила ничья
- Кнопка "Новая игра" сбрасывает всё в начало

### Механика игры

**Как определяется победитель:**
После каждого хода проверяются все 8 возможных выигрышных комбинаций (3 горизонтали, 3 вертикали, 2 диагонали). Если три клетки из одной комбинации заполнены одинаковым символом - этот игрок победил

**Как обрабатываются клики:**
При клике на клетку проверяется - пустая ли она и активна ли игра. Если да - в массив board записывается символ текущего игрока, клетка отображает символ, затем вызывается проверка результата

**Как работает рестарт:**
Функция restartGame сбрасывает currentPlayer на X, очищает массив board, ставит gameActive в true и убирает все символы с клеток

### Структура данных

**Массив board:**
Массив из 9 элементов, каждый соответствует клетке поля. Индексы от 0 до 8 идут слева направо сверху вниз. Пустая строка означает свободную клетку, "X" или "O" - занятую

**Выигрышные комбинации (winConditions):**
Массив из 8 вложенных массивов с индексами клеток. Например [0,1,2] - первая горизонталь, [0,3,6] - первая вертикаль, [0,4,8] - диагональ

**Хранение состояния игры:**
- currentPlayer - чей сейчас ход
- board - логическое состояние поля
- gameActive - флаг активности игры (false когда игра окончена)

### Алгоритмы

**Проверка победителя:**
Цикл перебирает все 8 комбинаций. Для каждой берёт три индекса и проверяет: все три клетки не пустые и равны между собой. Если находит такую комбинацию - устанавливает флаг победы

**Определение ничьей:**
После проверки на победу смотрит есть ли в массиве board хоть одна пустая строка. Если пустых нет - ничья

**Смена игрока:**
Тернарный оператор: если currentPlayer равен "X" - меняет на "O", иначе на "X"
